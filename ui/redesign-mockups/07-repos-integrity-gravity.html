what the repos tab is really for

Integrity and gravity — the two questions a list can't answer.

✓Mapping integrity

Three tiers: tracked (resolves), registered-not-local (pointer-only — map a clone), shadow (referenced but unregistered). The health line calls out the gap. No more silent "not registered" surprises.

◇Shadow repos surfaced

A repo peers --ref but the inventory doesn't know = a ghost row you can register in one step. The gap becomes a to-do, not a silent warning.

▁▃▅Reference gravity

Each repo shows how much it's talked about (refs + a density spark). You see what the fleet's work actually centers on — confer is hot, others quiet.

◈Drill-in composes cards

Open a repo → its hot files ranked by conversation density (mini code-ref cards → the big code view). Repos → Code is one continuous thread.

⌛Actions — deferred

Map / register / pull are the fix-the-gap moves — but serve is read-only today, so they wait on a future human-action layer. Kept in the design as intent (dashed, inactive); the read-only view still surfaces every gap now.

✓Honest by construction

Registration, clone-map, access, and ref counts are all real (inventory + repos.json + reverse index). Shadow detection is a pure diff. Nothing invented.
