what Full density adds back

Flip to Full and the same rows expand to the complete message — rendered markdown, inline refs anchored in the prose, and the actual code blocks in the code view. Summary is for scanning; Full is for reading in place; f is for reading one deeply.

the glanceability moves

Orientation in time and space, by construction.

≡Sticky day header

The day divider pins to the top as you scroll — "Today · Jul 18" is always visible. You never have to scroll back to find out which day you're in. One divider, not a date stamped on every note.

—Summary means summary

One line + compact chips (◆ refs, ⟨⟩ code). No rendered code blocks, no ref cards in summary — those are for Full. The stream becomes scannable in one pass instead of a column of shrunk full messages.

◈The NEW line = direction

The "new since you last looked" marker (real, from your local watermark) sits between what you've seen and what's fresh — so up = read, down = new, at a glance.

·Chips carry the density

`◆ 6` refs, `⟨⟩ 1` code block — you see a message HAS code/refs without it taking over the row. Click the chip (or open Full / focus) to see them.

›Consistent prev/next

The reader's prev/next borrows the minimap's dot+kind vocabulary, so moving through a thread feels like moving on the map — one visual language across the whole app.

🕗Local time everywhere

Already shipped: every timestamp is your local clock (5:09 PM, not 23:09 UTC), so "when" is legible too — the temporal-orientation groundwork.
