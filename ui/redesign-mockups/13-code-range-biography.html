Why it's a distinct view, not the thread view. The thread view is topic-first — a channel's stream, code embedded as refs. This is subject-first: one code range as the spine, its whole cross-thread discussion + version history, with the diffs the discussion drove. Recommendation: don't build it today — the file view's sidebar timeline (piece 12) is its MVP and its entry point ("expand this range →"). We graduate to this full view later, if a hotly-revised range proves the need. Reuses everything we already have: refs pin code at shas, patches carry the diffs, the composable message/code cards render the entries.
