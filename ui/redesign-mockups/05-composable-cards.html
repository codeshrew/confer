A list row in a stream, a full popover for detail, and a rich mini card that embeds inside other views and portals into the full one. why one vocabulary of cards

The mini card is the connective tissue.

▭Three sizes, one identity

Row to scan, mini to embed, full to inspect — same ticket, same colors, same lifecycle language. You always recognize it.

◈Mini = a portal

A mini card is a rich preview and a link. It shows progress + who + age, and opening it lands you in the full view — never a dead end.

⌗Already the pattern

CodeRefCard is the mini code view that leads to the big code view. We just generalize it — tickets and notes get the same treatment.

↔Everything relates to everything

A note touches tickets, code, a thread. The related column makes those first-class, selectable mini cards — not buried links.

jkKeyboard all the way down

The related column is a focusable pane: ↑↓/j k select a mini card, ↵ opens its full view. Composition is navigable.

→The path to a richer thread view

This is how the note/thread popover gets rich: its body, then its related tickets · code · thread as mini cards. Build the vocabulary, the rich views compose themselves.
